Processor Architecture II

SEQ: Sequential Implementation

Building Blocks

  • Computable Boolean
  • Continuously respond to input
  • Operate on data and implement control

Storage Elements

  • Store
  • Addressbale memories and NON-addressable registers
  • Clock rises -> load

SEQ Hardware Structure

Program counter register (PC)

Condition code register (CC)

Algorithm logical unit (ALU)

SEQ Stages

For detailed information, go to P250

  • Fetch: Read instruction from instruction memory
  • Decode
  • Excute: Compute(ALU)
  • Memory: I/O
  • Write back: Write PC
  • PC: Update PC

Instruction Decoding

Format

  • Instruction byte: icode:ifun
  • Optional register byte: rA:rB
  • Optional constant word: valC

Executing Arith./Logical Operation

OPL

OPL rA, rB							6 fn rA rB

functionsWhat to doWhat do we get
FetchRead 2 bytesicode:ifun, rA:rB, valP
DecodeRead operand registers valA, valB
ExecutePerform ALU operation and set conditon codesvalE, Set CC
MemoryNothing
Write backUpdate registerR[rB]
PC updateIncrement PC by 2PC <- valP

functionsWhat to doWhat do we get

Write backDo nothing
PC updateIncrement PC by 6PC <- valP

popl

popl rA    		b 0 rA F

functionsWhat to doWhat do we get
FetchRead 2 bytesicode:ifun, rA:rB,valP
DecodeRead stack pointervalA, valB
ExecuteIncrement stack pointer by 4valE <- valB+4
MemoryRead from stackvalM <-(valA)
Write backUpdate stack pointer and write result to registerR[%rsp], R[rA]
PC updateIncrement PC by 2PC <- valP

Must update popped value and new stack pointer

Jumps

jXX Dest    	7 fn Dest

functionsWhat to doWhat do we get
FetchRead 5 bytes and increment PC by 5icode:ifun, valC,valP
DecodeDo nothing
ExecuteDetermine which branch to takeCnd
MemoryDo nothing
Write backDo nothing
PC updateSet PC to proper destinationPC <- Cnd ? valC : valP

valC: target address valP: fall through address
